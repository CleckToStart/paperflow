做一个**轻量级本地工作流工具**，不改 OpenCode，只用“文件 + Prompt + 脚本”。

## 方案定位

项目名可以叫：

```text
paperflow
```

目标不是多 agent 平台，而是：

```text
输入论文/代码/实验结果
→ 分阶段审查
→ 输出修改报告、风险清单、修订计划
```

## 最小目录结构

```text
paperflow/
├── input/
│   ├── paper.tex
│   ├── figures/
│   └── results/
├── prompts/
│   ├── 01_structure_review.md
│   ├── 02_logic_review.md
│   ├── 03_experiment_review.md
│   ├── 04_language_review.md
│   └── 05_final_summary.md
├── output/
│   ├── 01_structure_review.md
│   ├── 02_logic_review.md
│   ├── 03_experiment_review.md
│   ├── 04_language_review.md
│   └── final_report.md
├── workflow.yaml
└── run.py
```

## 第一版工作流

```yaml
input: input/paper.tex

steps:
  - name: structure_review
    prompt: prompts/01_structure_review.md
    output: output/01_structure_review.md

  - name: logic_review
    prompt: prompts/02_logic_review.md
    output: output/02_logic_review.md

  - name: experiment_review
    prompt: prompts/03_experiment_review.md
    output: output/03_experiment_review.md

  - name: language_review
    prompt: prompts/04_language_review.md
    output: output/04_language_review.md

  - name: final_summary
    prompt: prompts/05_final_summary.md
    depends_on:
      - output/01_structure_review.md
      - output/02_logic_review.md
      - output/03_experiment_review.md
      - output/04_language_review.md
    output: output/final_report.md
```

## 每个阶段做什么

### 1. 结构审查

检查：

```text
摘要是否完整
引言是否有问题-方法-贡献闭环
相关工作是否支撑创新点
方法部分是否可复现
实验部分是否和主张对应
结论是否夸大
```

### 2. 逻辑审查

检查：

```text
创新点是否真实成立
前后表述是否矛盾
方法动机是否牵强
Loss 改进是否有数学或实验支撑
边缘设备友好叙事是否闭环
```

### 3. 实验审查

检查：

```text
baseline 是否足够
ablation 是否完整
指标是否合适
速度、参数量、FLOPs 是否支撑轻量化主张
结果是否存在选择性汇报
实验结论是否超过数据支持范围
```

### 4. 语言审查

检查：

```text
SCI 表达是否自然
术语是否统一
句子是否冗长
是否有中式英语
是否存在过度宣传式表达
```

### 5. 总结报告

输出：

```text
严重问题
中等问题
轻微问题
必须修改项
建议修改项
投稿风险
优先级排序
```

## 技术实现

第一版用 Python 就够。

核心逻辑：

```python
import os
import yaml
from pathlib import Path
from openai import OpenAI

client = OpenAI()

def read_file(path):
    return Path(path).read_text(encoding="utf-8")

def write_file(path, text):
    Path(path).parent.mkdir(parents=True, exist_ok=True)
    Path(path).write_text(text, encoding="utf-8")

def run_step(step, main_input):
    prompt = read_file(step["prompt"])

    extra_context = ""
    for dep in step.get("depends_on", []):
        extra_context += f"\n\n===== Previous Report: {dep} =====\n"
        extra_context += read_file(dep)

    user_content = f"""
你正在审查一篇SCI论文。

===== 审查任务 =====
{prompt}

===== 论文全文 =====
{main_input}

{extra_context}
"""

    response = client.chat.completions.create(
        model="gpt-4.1",
        messages=[
            {"role": "system", "content": "你是一名严格的SCI论文审稿专家，专长于目标检测、YOLO、轻量化网络与Loss函数改进。"},
            {"role": "user", "content": user_content}
        ],
        temperature=0.2,
    )

    return response.choices[0].message.content

def main():
    config = yaml.safe_load(read_file("workflow.yaml"))
    main_input = read_file(config["input"])

    for step in config["steps"]:
        result = run_step(step, main_input)
        write_file(step["output"], result)
        print(f"Finished: {step['name']}")

if __name__ == "__main__":
    main()
```

## 第一版不要做的东西

先不要做：

```text
Web UI
多模型调度
OpenCode 源码改造
插件系统
数据库
复杂记忆系统
并行 agent
自动改全文
```

这些都会让工具变重。

## 最实用的 MVP

你的第一版只需要做到：

```text
输入 paper.tex
运行 run.py
输出 final_report.md
```

够了。

后续再加：

```text
代码实验检查
LaTeX 编译检查
图表编号一致性检查
参考文献真实性检查
重复表述风险检查
自动生成 response letter
```

最关键的一句话：

**先做“串行审稿流水线”，不要做“多 agent 平台”。**
